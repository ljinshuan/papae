## Context

本项目为全新开发，无现有代码库需要适配。目标用户是婴幼儿家长，通过录制宝宝走路视频获得初步姿态评估。系统核心挑战在于：(1) 婴幼儿体型小、步态不规律，传统 CV 模型检测效果可能不佳；(2) 需要兼顾 LLM 视觉理解能力与 token 成本控制；(3) 需处理多种边界情况（视频过短、模糊、多人物等）。

## Goals / Non-Goals

**Goals:**
- 实现一个命令行工具，输入视频、输出评估报告和可视化视频
- 本地完成 CV 推理和关键帧提取，只将筛选后的关键帧传给 LLM
- 支持 YOLO-pose + YOLO-seg 联合推理，输出骨架和分割叠加效果
- 基于脚踝轨迹检测步态周期，提取 4 个关键相位帧
- 生成结构化的 Markdown 报告（风险等级、发现、建议）

**Non-Goals:**
- 实时摄像头分析（仅支持预录制视频）
- 医学诊断（明确标注"仅供参考"）
- 本地部署 LLM（当前使用云端 API）
- 移动端 APP 或 Web 界面
- 多语言支持（仅中文）
- 用户账户系统或历史记录管理

## Decisions

### 1. 架构模式：流水线（Pipeline）
**决策：** 采用单向数据流流水线架构，7 个组件依次执行。  
**理由：** 各阶段职责清晰、易于独立测试；数据流可预测，便于调试和错误定位。  
**替代方案：** 事件驱动架构（过度设计，无并发需求）、DAG 编排（如 Prefect/Airflow，引入不必要的复杂度）。

### 2. 关键帧策略：步态周期提取 + 退化采样
**决策：** 优先基于脚踝 Y 坐标轨迹检测步态周期，提取 4 个关键相位帧；检测失败时退化为均匀采样 8 帧。  
**理由：** 大幅减少 LLM 输入量（从数百帧降至 8-20 帧），同时保留医学上重要的步态相位信息。  
**替代方案：** 逐帧全量分析（token 成本不可接受）、固定间隔采样（可能错过关键相位）。

### 3. LLM 角色：端到端视觉判断
**决策：** 将关键帧图像 + 姿态数据直接发送给全模态 LLM，由模型自行判断异常，不预设医学阈值规则。  
**理由：** 全模态模型具备视觉理解能力，可捕捉人眼/规则难以定义的姿态异常模式；避免硬编码规则在不同月龄适用性问题。  
**替代方案：** 规则引擎 + LLM 辅助（规则维护成本高，婴幼儿发育阶段差异大）。

### 4. YOLO 选型：Ultralytics YOLOv8/v11
**决策：** 使用 ultralytics 框架的 YOLO-pose 和 YOLO-seg 模型。  
**理由：** 统一框架简化部署，模型尺寸可选（n/s/m/l），生态成熟；supervision 库原生兼容。  
**替代方案：** MediaPipe Pose（不支持分割）、RTMPose（需额外部署）、MMPose（配置复杂）。

### 5. 包管理：uv
**决策：** 使用 uv 替代 pip/poetry 进行包管理和虚拟环境。  
**理由：** 极速依赖解析和安装，支持 PEP 621 标准 pyproject.toml，lock 文件确保可复现。  
**替代方案：** poetry（安装慢，非标准 pyproject.toml）、pip + requirements.txt（无 lock 文件）。

### 6. 类型检查：basedpyright
**决策：** 使用 basedpyright 替代 mypy 进行静态类型分析。  
**理由：** 基于 pyright，速度更快，与 VS Code/PyCharm LSP 集成好；basedpyright 是社区维护的增强版。  
**替代方案：** mypy（较慢，配置复杂）、pyright（功能较少）。

### 7. 数据模型：Pydantic v2
**决策：** 使用 Pydantic v2 定义所有组件间传递的数据结构。  
**理由：** 类型安全、序列化友好、与 basedpyright 配合良好；pydantic-settings 可用于 CLI 配置。  
**替代方案：** dataclasses（无验证）、TypedDict（功能有限）。

## Risks / Trade-offs

| 风险 | 缓解措施 |
|-----|---------|
| YOLO 对婴幼儿小体型检测效果差 | 提供模型尺寸选择（n/s/m/l），置信度阈值可调；必要时准备婴幼儿数据微调 |
| LLM 判断不准确或幻觉 | 明确免责声明；保留原始响应供人工复核；后续可引入专家标注数据微调 |
| 步态周期检测对不规则步态失效 | 退化策略（均匀采样）；后续探索基于学习的相位检测 |
| 视频拍摄角度/光线影响检测 | 在报告和 README 中提供拍摄建议（侧面、全身、光线充足、地面平坦） |
| 云端 LLM API 延迟或不可用 | 指数退避重试；超时后保留中间结果并清晰报错 |
| 多人物场景误检 | 只保留最大 bbox 的人形；在报告中备注 |

## Migration Plan

本项目为全新项目，无迁移需求。首次使用需：
1. 安装 uv：`curl -LsSf https://astral.sh/uv/install.sh | sh`
2. 创建虚拟环境：`uv venv --python 3.13`
3. 安装依赖：`uv pip install -e ".[dev]"`
4. 下载 YOLO 模型权重（首次运行时自动下载或手动放置）
5. 配置 LLM API 密钥
